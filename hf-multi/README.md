# HF Multi

Консольное приложение, которое **параллельно** вызывает несколько моделей
HuggingFace на один промпт и печатает их ответы рядом — с обозначением модели
и ссылкой на неё. На общем ядре [`../core`](../core).

## Как выбираются модели

По умолчанию приложение запрашивает у HuggingFace Hub популярные chat-модели
(`text-generation`), оставляет те, у кого известно число параметров и есть
провайдер инференса, сортирует по числу параметров и берёт **три**: самую
крупную, среднюю и самую маленькую.

Набор можно задать вручную — флагом `--models` или интерактивно при запуске.

Промпт можно передать флагом `--prompt` — тогда приложение работает
**неинтерактивно**: не спрашивает ни смену моделей, ни промпт, а сразу
запрашивает модели и печатает ответы (удобно для скриптов и пайпов).

## Требования

- Node.js **>= 22.6**, каталог `../core` рядом.
- Свой `.env` (см. `.env.example`): токен HuggingFace `hf_…` в `LLM_API_KEY`,
  `LLM_BASE_URL=https://router.huggingface.co/v1/chat/completions`. `LLM_MODEL`
  обязателен по схеме конфигурации, но приложением не используется (модели
  выбираются отдельно) — поставьте любую валидную.

## Установка и запуск

```bash
npm install
cp .env.example .env   # впишите hf_-токен

# модели по умолчанию (крупная/средняя/мелкая по параметрам)
npm start

# свой набор моделей
npm start -- --models="Qwen/Qwen2.5-7B-Instruct,meta-llama/Meta-Llama-3-8B,Qwen/Qwen2.5-0.5B"

# неинтерактивно: промпт сразу в опции
npm start -- --models="Qwen/Qwen2.5-Coder-7B-Instruct:nscale" --prompt="Объясни рекурсию"
```

Приложение покажет выбранные модели, предложит сменить их (Enter — оставить),
спросит промпт и выведет ответы всех моделей параллельно:

```
### moonshotai/Kimi-K2-Instruct-0905 — 1026.5 B
https://huggingface.co/moonshotai/Kimi-K2-Instruct-0905
время: 2.3 c · токены: вход 45, выход 48, всего 93

<ответ модели>
────────────────────────────────────────────────────────────
### Qwen/Qwen2.5-0.5B — 494 M
https://huggingface.co/Qwen/Qwen2.5-0.5B
время: 5.1 c · токены: вход 45, выход 512, всего 557
...
```

Под ссылкой выводится строка метрик: **время ответа** и **число токенов**
по отдельности — **вход** (промпт), **выход** (ответ) и **всего**; если провайдер
не прислал usage — `н/д`.

Ошибка одной модели (недоступна, таймаут) не роняет остальные — у неё в выводе
будет `[ошибка] …` (время при этом тоже показывается).

## Проверка типов и тесты

```bash
npm run typecheck
npm test            # node:test, покрытие 100% (строки/ветки/функции)
```

Отбор моделей (HF Hub) тестируется без токена; сама генерация требует `hf_…`.

## Структура

| Файл               | Назначение                                                  |
| ------------------ | ----------------------------------------------------------- |
| `src/hub.ts`       | Запрос к HF Hub, разбор и отбор моделей по числу параметров  |
| `src/generate.ts`  | Параллельные запросы к моделям и форматирование вывода       |
| `src/run.ts`       | Интерактивный сценарий (смена моделей + промпт)              |
| `src/multi-cli.ts` | Точка входа (конфиг, фабрика клиентов, выбор моделей)        |
