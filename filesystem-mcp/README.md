# filesystem-mcp

MCP-сервер **файловой системы** с «мягкой» песочницей (allow-list). Даёт агенту читать и писать
файлы и каталоги в разрешённых директориях; путь **вне** allow-list не отклоняется сразу, а
запрашивает подтверждение у пользователя через MCP **elicitation**.

> Изолированный пакет, **не часть ядра** `ai-advent`. Сознательно использует официальный
> `@modelcontextprotocol/sdk` + `zod` (как `yandex-ocr-mcp`/`places-mcp`). Нативный TS без сборки,
> 100% покрытие логики (тонкие `server.ts`/`cli.ts` вне покрытия).

## Инструменты

| Инструмент | Назначение |
| ---------- | ---------- |
| `read_file` | Прочитать текстовый файл. |
| `write_file` | Создать/перезаписать файл (родительские каталоги создаются). |
| `append_file` | Дописать в конец файла (создаёт, если нет). |
| `list_dir` | Список файлов и папок (по умолчанию — корень allow-list). |
| `delete_path` | Удалить **одиночный** файл или **пустую** папку. |

## Безопасность

- **Allow-list** — разрешённые каталоги задаются позиционными аргументами или `FS_ALLOWED_DIRS`
  (через запятую). Путь резолвится без отказа (`classifyPath`), затем проходит ворота
  `authorizePath`: внутри allow-list — сразу; вне — запрос подтверждения.
- **Подтверждение вне песочницы (elicitation)** — если клиент поддерживает elicitation (как
  `llm-cli`), сервер спрашивает «⚠ путь вне разрешённых каталогов, разрешить?»; пользователь
  отвечает да/нет. Клиент без elicitation → безопасный отказ (поведение жёсткой песочницы).
- **«Стереть всё» невозможно** — `delete_path` не делает рекурсивного удаления (непустую папку не
  трогает) и не удаляет сам корень allow-list. Эти защиты действуют **даже при подтверждении**.

## Запуск

```bash
npm install
# разрешённые каталоги — аргументами или через FS_ALLOWED_DIRS
node src/cli.ts ~/Documents ~/projects
# или: FS_ALLOWED_DIRS="~/Documents,~/projects" node src/cli.ts
```

## Подключение к `llm-cli`

```
/mcp add filesystem node /путь/к/filesystem-mcp/src/cli.ts ~
```

Песочница — домашний каталог (`~`); операции вне него `llm-cli` подтвердит у пользователя
(⚠ да/нет) через свой мост elicitation.

## Команды

```bash
npm run typecheck       # tsc
npm test                # node:test + покрытие 100/100/100 (server/cli исключены)
npm run write-prettier  # форматирование
```

## Структура

| Файл | Назначение |
| ---- | ---------- |
| `src/sandbox.ts` | Разворот `~`, нормализация allow-list, `classifyPath` (резолв без отказа). |
| `src/operations.ts` | Низкоуровневые ФС-операции (`FsIo`-шов + реализация на `node:fs`). |
| `src/config.ts` | Разбор разрешённых каталогов (аргументы / `FS_ALLOWED_DIRS`). |
| `src/tools.ts` | Обработчики инструментов + ворота `authorizePath` (подтверждение вне песочницы). |
| `src/server.ts` / `src/cli.ts` | Тонкая обвязка: `McpServer` (+ `confirm` через `elicitInput`) + stdio (вне покрытия). |
