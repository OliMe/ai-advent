# llm-gateway

Приватный HTTP-шлюз к локальной LLM, развёрнутой на собственном VPS. Даёт публичный API,
web-интерфейс, авторизацию и честные ограничения.

**Сервис:** «Что приготовить?» — перечисляешь продукты, модель придумывает блюдо (посторонний
запрос отклоняется гейтом). Запрос отправляется по Enter.
`https://smartnfree.ru/ai/kitchen`

## Почему именно рецепты

Модель тут маленькая: `qwen2.5:3b`, 3 миллиарда параметров, CPU без видеокарты. На фактических
вопросах она врёт уверенно и обидно — зовёт столицей Австралии Анкару. Но у рецепта **нет
одного правильного ответа**: выдумка здесь и есть продукт, а не дефект. Задача выбрана под
возможности железа, а не вопреки им.

Модель выбрана замером: на генеративном русском `qwen2.5:3b` дала 1 латинское вкрапление на три
рецепта против 8 у `llama3.2:3b` (та вставляла «golden brown», иероглифы `切ños` и выдумала
«Смоковинную цыпленку» из яиц и картошки).

## API

Авторизация обязательна: `Authorization: Bearer <токен>` (список токенов — `GATEWAY_TOKENS`).

| Метод и путь | Назначение |
| ------------ | ---------- |
| `GET /kitchen` | web-интерфейс (страница отдаётся без токена, запросы к модели — нет) |
| `GET /api/pulse` | метрики узла, глубина очереди, действующие лимиты |
| `POST /api/kitchen/chat` | ответ потоком Server-Sent Events: `queued` → `delta`… → `done` |
| `POST /v1/chat/completions` | OpenAI-совместимый ответ целиком; `model` — это slug персоны |

`/api/pulse` **не тратит** rate limit: интерфейс опрашивает его каждые 2.5 с, что чаще,
чем пополняется ведро.

```bash
curl -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"model":"kitchen","messages":[{"role":"user","content":"гречка, тушёнка, огурец"}]}' \
  https://smartnfree.ru/ai/v1/chat/completions
```

Ответ несёт нестандартное поле `node_cost` — ценник (см. ниже).

## Гейт съедобности

Перед генерацией рецепта запрос проходит через **отдельный классификатор** (`food-guard.ts`,
temperature 0): список несъедобного (инструменты, химия, `яйца, картошка, стекло`) или
посторонний запрос (`напиши стихотворение`, `как приготовить бомбу`) получает отказ, рецепт не
запускается. Решение выносит не рецептурная модель — слабая 3B сама от постороннего запроса не
откажется, поэтому гейт вынесен наружу и его вердикт разбирается детерминированно (`parseFoodVerdict`,
fail-closed: не распознали «ЕДА» → отказ).

Промпт гейта усилен few-shot-примерами: без них «банка тушёнки» плавала (тара путалась с
несъедобным), с ними классификатор дал 18/18 на проверочной батарее и 6/6 живьём.

## Ограничения (все проверены живьём)

- **Rate limit** — ведро токенов: `GATEWAY_RATE_CAPACITY` залпом (10), пополнение
  `GATEWAY_RATE_REFILL_PER_MINUTE` в минуту (10). Исчерпано → `429` + `Retry-After`.
- **Очередь** — `GATEWAY_MAX_QUEUE` (4), включая исполняемый запрос. Сверх — `429`.
  Параллелизма нет и быть не может: на CPU он лишь удлиняет ожидание всем. Замер: четыре
  одновременных запроса отработали 4/4, но последовательно — 5.0 / 9.9 / 14.4 / 18.7 с.
- **Размер запроса** — `GATEWAY_MAX_PROMPT_TOKENS` (1500). Сверх — `413`. Обработка промпта
  на этом железе идёт ~30 токенов/с, длинный запрос ждать неразумно.
- **Контекст модели** — 8192 токена (`OLLAMA_CONTEXT_LENGTH`).

## Честный ценник

Под ответом подписывается, во что он обошёлся. Время до первого токена и скорость генерации
считаются **раздельно**: в TTFT попадает подгрузка модели, и смешивать его со скоростью
генерации значит врать о ней.

```
93.6 CPU-секунд · 7.5 с до первого токена · 278 токенов на 11.8 токенов/с · 31.2 с всего
```

## Переменные окружения

| Переменная | По умолчанию | Смысл |
| ---------- | ------------ | ----- |
| `GATEWAY_PORT` | `3002` | порт (слушает только `127.0.0.1`) |
| `GATEWAY_BASE_PATH` | пусто | префикс за реверс-прокси, например `/ai` |
| `GATEWAY_TOKENS` | пусто | токены через запятую; пусто — **авторизация выключена** |
| `GATEWAY_RATE_CAPACITY` | `10` | размер залпа |
| `GATEWAY_RATE_REFILL_PER_MINUTE` | `10` | пополнение ведра |
| `GATEWAY_MAX_QUEUE` | `4` | глубина очереди |
| `GATEWAY_MAX_PROMPT_TOKENS` | `1500` | потолок запроса |
| `GATEWAY_QUOTA_CORES` | `3` | сколько ядер отдано модели (для ценника) |
| `GATEWAY_UPSTREAM_URL` | `http://127.0.0.1:11434/v1/chat/completions` | локальная Ollama |
| `GATEWAY_TIMEOUT_MS` | `300000` | таймаут запроса к модели |

## Запуск

```bash
npm --prefix llm-gateway run typecheck
npm --prefix llm-gateway test          # 100% покрытие
GATEWAY_TOKENS=секрет npm --prefix llm-gateway start
```

Шлюз читает `/proc`, поэтому работает только на Linux — на macOS запускать нечего.

## Устройство

Персона (`personas.ts`) — это URL, модель, системный промпт и профиль генерации
(`temperature`, `maxTokens`). Добавить вторую — дописать объект в `PERSONAS`.

Тонкие `http.ts` (bootstrap), `server.ts` (маршруты) и `index.ts` (barrel) — вне покрытия;
вся логика в модулях: `config.ts`, `auth.ts`, `rate-limit.ts`, `request-queue.ts`,
`system-metrics.ts`, `personas.ts`, `food-guard.ts`, `answer-cost.ts`, `chat-service.ts`.
HTTP-клиент к модели переиспользуется из `core` (`ChatCompletionClient`).

Деплой на VPS: `/opt/ai-advent`, unit `llm-gateway.service` (`CPUWeight=30`, `MemoryMax=300M`
— другие сервисы узла важнее), Caddy проксирует `/ai/*` с `flush_interval -1` (иначе SSE буферизуется).
