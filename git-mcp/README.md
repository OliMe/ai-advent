# git-mcp

MCP-**сервер** доступа к git-репозиториям **только на чтение**: текущая ветка, статус, список
файлов, история, diff, точный поиск по коду и чтение файла. Через него ассистент разработчика
понимает *структуру и состояние* любого проекта — того, что не написано в документации.

> Изолированный пакет, не часть ядра `ai-advent`. Осознанное исключение из правил репозитория:
> использует официальный `@modelcontextprotocol/sdk` + `zod`. Нативный TypeScript без сборки, 100%
> покрытие логики (тонкие `server.ts`/`cli.ts` — только проводка к SDK, вне покрытия).

## Инструменты

| Инструмент | Аргументы | Что делает |
| ---------- | --------- | ---------- |
| `git_branch` | `repo?` | Текущая ветка (или отделённый HEAD с коммитом) |
| `git_status` | `repo?` | Изменённые и неотслеживаемые файлы (короткий статус) |
| `git_list_files` | `repo?`, `subdir?` | Отслеживаемые файлы — структура проекта |
| `git_log` | `repo?`, `limit?` (1–50, дефолт 10) | Последние коммиты: хэш, дата, автор, заголовок |
| `git_diff` | `repo?`, `path?`, `staged?` | Незакоммиченные изменения (рабочее дерево или индекс) |
| `git_grep` | `repo?`, `pattern`, `subdir?` | Точный поиск по коду (`git grep`) |
| `read_file` | `repo?`, `path` | Файл рабочего дерева репозитория |

`repo` — путь к репозиторию; без него берётся первый разрешённый. **Один сервер обслуживает
несколько проектов**: при нескольких привязанных проектах агент указывает `repo` явно.

## Безопасность

- **Только чтение.** Пишущих подкоманд git нет вовсе.
- **Аргументы git — фиксированный массив**, строка от модели в `argv` не попадает: подставить свой
  флаг или подкоманду невозможно. Запуск без шелла (`execFileSync`).
- **Мягкая песочница** (как в `filesystem-mcp`): репозиторий вне allow-list не отклоняется сразу —
  через MCP **elicitation** у пользователя спрашивают подтверждение (⚠ да/нет). Без обработчика
  подтверждений — отказ. Так ассистент работает с любым проектом, не требуя правки конфигурации.
- **Путь наружу репозитория** (`../../etc/passwd`, абсолютный путь вне корня) не разрешается ничем:
  это уже не операция над репозиторием.
- **Вывод усекается** (`GIT_MAX_OUTPUT_CHARS`, дефолт 8000 символов) с честной пометкой об
  усечении — гигантский diff не съедает окно контекста и не выдаёт обрывок за целое.

## Запуск

```bash
npm install
npm start -- /путь/к/репозиторию [/другой/репозиторий …]   # stdio
```

Разрешённые репозитории — **объединение** источников, а не «или-или»:

| Источник | Кто задаёт |
| -------- | ---------- |
| Позиционные аргументы | Пользователь вручную (в `mcp.json`) |
| `GIT_ALLOWED_REPOS` (через запятую) | `llm-cli` — сюда он дописывает привязанные проекты (`/project add`) |
| Рабочий каталог сервера | Всегда: сервер запущен там, откуда его позвали — это «текущий» проект |
| `~/.llm-cli/projects` (кэш клонов) | Всегда: репозитории туда клонирует сам ассистент |

Объединение — намеренно: клиент, добавляя проект, не должен незаметно **отбирать** доступ к тому,
что уже работало. Порядок тоже значим — первый элемент служит репозиторием **по умолчанию** (вызов
без `repo`) и базой для относительных путей, поэтому ручная настройка и рабочий каталог идут раньше
автоматически добавленных проектов: иначе привязка чужого клона молча сделала бы его дефолтом.

## Подключение к `llm-cli`

```
/mcp add git node /Users/<вы>/projects/ai-advent/git-mcp/src/cli.ts ~/projects
```

После этого ассистент отвечает на вопросы о состоянии репозитория («на какой ветке проект?», «что
сейчас изменено?») и достаёт код, которого нет в документации (`git_grep`, `read_file`).

## Структура

| Файл | Назначение |
| ---- | ---------- |
| `src/operations.ts` | Шов `GitIo`: запуск git (без шелла) + чтение рабочего дерева; ненулевой код — `ok: false`, не исключение |
| `src/sandbox.ts` | Allow-list: резолв пути без отказа (`classifyPath`) и запрет выхода за корень репо (`resolveInsideRepo`) |
| `src/config.ts` | Разрешённые репозитории (аргументы/`GIT_ALLOWED_REPOS`/cwd + кэш клонов), потолок вывода |
| `src/tools.ts` | Обработчики инструментов: ворота доступа → фиксированная git-команда → текст |
| `src/server.ts` | Проводка к MCP SDK: `registerTool` + zod-схемы, elicitation-подтверждение |
| `src/cli.ts` | Точка входа (stdio) |
