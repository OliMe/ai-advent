# mcp-probe

Пробный MCP-клиент для исследования готовых MCP-серверов. Подключается к серверу
**Tavily** (поиск в интернете), запрашивает список инструментов и печатает ответ в консоль.

> Изолированная песочница, **не часть ядра** `ai-advent`. Здесь сознательно используется
> официальный `@modelcontextprotocol/sdk` (собирать MCP по stdio вручную нет смысла), что
> отличается от соглашения остальных пакетов «без сторонних SDK». Остальные договорённости
> соблюдены: нативный TS без сборки, модульность, `tsconfig`/prettier, 100% покрытие логики.

## Установка

```bash
npm --prefix mcp-probe install     # из корня репозитория
# или: cd mcp-probe && npm install
```

## Запуск

Нужен ключ Tavily (получить на tavily.com → API key). Запуск **из каталога пакета**:

```bash
cd mcp-probe
TAVILY_API_KEY=tvly-xxxxx npm run list-tools
```

Клиент поднимет сервер `tavily-mcp` через `npx` по stdio, подключится и выведет JSON со
списком инструментов сервера (имена, описания, схемы входных параметров).

## Структура

| Файл              | Назначение                                                          |
| ----------------- | ------------------------------------------------------------------- |
| `src/tavily.ts`   | Чистая логика: ключ из окружения, окружение и параметры запуска сервера |
| `src/probe.ts`    | `probeTools`: connect → listTools → close (узкий контракт `ToolProbe`) |
| `src/index.ts`    | Barrel — реэкспорт логики (отсюда импортируют тесты)                 |
| `src/cli.ts`      | Тонкая точка входа: проводка к реальному SDK, печать JSON            |

## Дальше

Отсюда легко добавить вызов конкретного инструмента, например поиска:

```ts
const result = await client.callTool({
  name: 'tavily-search',
  arguments: { query: 'что такое Model Context Protocol' },
});
console.log(JSON.stringify(result, null, 2));
```
