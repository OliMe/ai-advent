# mcp-probe

Пробный MCP-клиент для исследования **любых** готовых MCP-серверов. Подключается к серверу
(по **stdio** или **Streamable HTTP**), настраивается через `.env`, и либо печатает список
инструментов сервера, либо вызывает указанный инструмент и печатает результат.

> Изолированная песочница, **не часть ядра** `ai-advent`. Здесь сознательно используется
> официальный `@modelcontextprotocol/sdk` (собирать MCP по stdio/HTTP вручную нет смысла), что
> отличается от соглашения остальных пакетов «без сторонних SDK». Остальные договорённости
> соблюдены: нативный TS без сборки, модульность, `tsconfig`/prettier, 100% покрытие логики.

## Установка

```bash
cd mcp-probe && npm install
cp .env.example .env   # заполните под свой сервер
```

## Настройка (`.env`)

| Переменная          | Назначение                                                                 |
| ------------------- | -------------------------------------------------------------------------- |
| `MCP_TRANSPORT`     | `stdio` или `http`. Необязательна — выводится из `MCP_URL`/`MCP_COMMAND`.   |
| `MCP_COMMAND`       | (stdio) команда запуска сервера, напр. `npx`.                              |
| `MCP_ARGS`          | (stdio) аргументы через пробел, напр. `-y tavily-mcp`.                      |
| `MCP_URL`           | (http) адрес Streamable HTTP, напр. `http://localhost:3000/mcp`.           |
| `MCP_BEARER_TOKEN`  | (http) токен → заголовок `Authorization: Bearer …`.                        |
| `MCP_HEADERS`       | (http) доп. заголовки: `Имя: значение`, пары через `;`.                    |
| `MCP_TOOL`          | Запасной (если не передавать инструмент аргументом CLI) — какой инструмент вызвать. |
| `MCP_TOOL_ARGS`     | Запасные аргументы вызова инструмента — JSON-объект.                       |

Для stdio-серверов всё окружение (включая `.env`) пробрасывается процессу сервера — туда же
кладите его API-ключи (напр. `TAVILY_API_KEY`).

## Запуск

Инструмент для вызова передаётся **первым аргументом командной строки**, его аргументы —
вторым (JSON-объект). Без аргумента выводится **список инструментов**. CLI-аргумент важнее
`MCP_TOOL`/`MCP_TOOL_ARGS` из `.env`.

```bash
npm start                                      # список инструментов сервера
npm start -- tavily-search                     # вызвать инструмент без аргументов
npm start -- tavily-search '{"query":"MCP"}'   # вызвать с аргументами
```

> `--` отделяет аргументы npm от аргументов скрипта. Через `node` напрямую: `node src/cli.ts tavily-search '{"query":"MCP"}'`.

Пример настройки сервера в `.env` (Tavily по stdio):

```bash
MCP_COMMAND=npx
MCP_ARGS=-y tavily-mcp
TAVILY_API_KEY=tvly-xxxxx
# HTTP-сервер вместо stdio:
# MCP_URL=http://localhost:3000/mcp
# MCP_BEARER_TOKEN=xxxxx
```

## Структура

| Файл             | Назначение                                                              |
| ---------------- | ----------------------------------------------------------------------- |
| `src/config.ts`  | Чистая логика: разбор `MCP_*` → конфиг транспорта и действие (`ProbeAction`) |
| `src/probe.ts`   | `runProbe`: connect → list/call → close (узкий контракт `McpProbe`)      |
| `src/index.ts`   | Barrel — реэкспорт логики (отсюда импортируют тесты)                     |
| `src/cli.ts`     | Тонкая точка входа: `.env`, выбор транспорта SDK, печать JSON            |
