# Настройка

## Как настроить эмбеддинги для RAG?

Эмбеддинги задаются переменными `LLM_EMBEDDINGS_URL` (эндпоинт `/embeddings`),
`LLM_EMBEDDINGS_MODEL` (модель, напр. `nomic-embed-text`) и `LLM_EMBEDDINGS_API_KEY` (если эндпоинт
за токеном). Без них RAG по документации мягко деградирует на сырые доки. Заголовок `Authorization`
добавляется только при заданном ключе — локальная Ollama работает без него.

## Как подключить MCP-сервер?

Команда `/mcp add <имя> <команда|URL>` подключает сервер и записывает его в `~/.llm-cli/mcp.json`.
Для stdio-сервера укажите команду запуска (`node .../cli.ts`), для HTTP — URL с заголовком
`Authorization`. `/mcp` показывает подключённые серверы, `/mcp remove <имя>` отключает,
`/mcp reload` перечитывает конфиг. Инструменты сервера появляются под именем `сервер__инструмент`.

## Как запустить локальную модель?

Установите Ollama и скачайте модель (`ollama pull <модель>`). Модель слушает `localhost:11434` и
совместима с OpenAI API. Укажите `LLM_BASE_URL=http://localhost:11434/v1` и `LLM_MODEL=<модель>`.
Учтите: локальная модель доступна только с той же машины — с облачного CI-раннера `localhost`
недоступен, там нужен облачный эндпоинт или VPS-шлюз.

## Как включить структурированный вывод этапов пайплайна?

Переменная `LLM_STRUCTURED_OUTPUTS=1` включает `response_format: json_schema` на этапах. По умолчанию
выключено — не все провайдеры это поддерживают (z.ai/GLM ломается на `response_format`). Включайте
только под модель, которая это умеет (проверено на Ollama).
