# ai-advent

Набор консольных приложений на Node.js + TypeScript для работы с LLM через
**OpenAI-совместимый** API (запросы — нативным `fetch`, без сторонних SDK).
Репозиторий организован как несколько независимых пакетов вокруг общего ядра.

## Пакеты

| Пакет | Назначение |
| ----- | ---------- |
| [`core/`](core) | Общее ядро: HTTP-клиент к OpenAI-совместимому API, загрузка конфигурации из окружения, типы. Сюда же — ретраи с экспоненциальным бэкоффом (429/5xx/сетевые сбои) и отключение «рассуждений» (`disableThinking`). Остальные пакеты импортируют его напрямую. |
| [`llm-cli/`](llm-cli) | Консольный чат с моделью: одиночный запрос и интерактивный режим, слоистая память, мультипрофили. Задачи исполняются автономным пайплайном (`/run`): сбор требований → планирование → выполнение → проверка → завершение. Флаги ограничений генерации — `--max-tokens`, `--stop`, `--json`, `--json-schema`, `--no-thinking`. |
| [`contract-extractor/`](contract-extractor) | Извлечение реквизитов сторон из договоров аренды. Два CLI: `extract-json` (строгий JSON-массив) и `list-text` (человекочитаемый список). |
| [`contract-generator/`](contract-generator) | Генерация больших списков синтетических договоров аренды (для тестовых данных). Режим стоп-маркера и пакетный режим. |
| [`task-solver/`](task-solver) | Решение одной задачи четырьмя способами промптинга (простой, пошаговый, двухшаговый, панель экспертов) и оценка GLM, какое решение точнее. |
| [`hf-multi/`](hf-multi) | Параллельный вызов нескольких моделей HuggingFace на один промпт (по умолчанию — крупная, средняя и мелкая по числу параметров) и сравнение ответов. |

Связка «генератор → экстрактор» образует замкнутый цикл: `contract-generator`
пишет файл договоров, который напрямую читает `contract-extractor`.

## Требования

- Node.js **>= 22.6** (нативный запуск `.ts` и `fetch`; зависимостей в рантайме нет).
- Доступ к OpenAI-совместимому API: URL, имя модели, API-ключ (по умолчанию
  настроено на z.ai / GLM).

## Быстрый старт

Каждый прикладной пакет — самостоятельный: установка и `.env` свои.

```bash
cd llm-cli            # или contract-extractor / contract-generator
npm install
cp .env.example .env  # впишите LLM_API_KEY, LLM_BASE_URL, LLM_MODEL
npm start             # см. README конкретного пакета
```

`core/` отдельно ставить не нужно — он подключается из соседних пакетов по
относительному пути. Свои зависимости (`@types/node`, `typescript`, `prettier`)
есть у каждого пакета.

## Конфигурация (общие переменные окружения)

Читаются ядром из `.env` в каталоге запуска (переменные окружения имеют приоритет):

| Переменная | Назначение | По умолчанию |
| ---------- | ---------- | ------------ |
| `LLM_API_KEY` | Ключ доступа к API | — (обязательно) |
| `LLM_BASE_URL` | URL эндпоинта chat/completions | — (обязательно) |
| `LLM_MODEL` | Имя модели | — (обязательно) |
| `LLM_TEMPERATURE` | Температура | `0.7` |
| `LLM_SYSTEM_PROMPT` | Системный промпт | нейтральный |
| `LLM_REQUEST_TIMEOUT_MS` | Таймаут запроса, мс | `60000` |
| `LLM_MAX_RETRIES` | Число повторов при 429/5xx/сбоях | `3` |
| `LLM_RETRY_BASE_MS` | Базовая задержка бэкоффа, мс | `500` |
| `LLM_STAGE_MAX_TOKENS` | Потолок генерации агентов пайплайна задач, токенов | дефолт провайдера |

## Разработка

В каждом пакете доступны одинаковые команды:

```bash
npm run typecheck   # проверка типов (tsc --noEmit)
npm test            # node:test, покрытие 100% (строки/ветки/функции)
npm run lint-prettier   # проверка форматирования
```

```
└── contract-generator/    генерация синтетических договоров
```

Подробности по флагам и примерам — в README соответствующего пакета.
