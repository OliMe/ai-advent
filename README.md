# ai-advent

Набор консольных приложений на Node.js + TypeScript для работы с LLM через
**OpenAI-совместимый** API (запросы — нативным `fetch`, без сторонних SDK).
Репозиторий организован как несколько независимых пакетов вокруг общего ядра.

## Пакеты

| Пакет | Назначение |
| ----- | ---------- |
| [`core/`](core) | Общее ядро: HTTP-клиент к OpenAI-совместимому API, загрузка конфигурации из окружения, типы. Сюда же — ретраи с экспоненциальным бэкоффом (429/5xx/сетевые сбои) и отключение «рассуждений» (`disableThinking`). Остальные пакеты импортируют его напрямую. |
| [`llm-cli/`](llm-cli) | Консольный чат с моделью: одиночный запрос и интерактивный режим, слоистая память, мультипрофили, инварианты. Задачи исполняются автономным пайплайном (`/run`): сбор требований → планирование → выполнение → проверка → завершение. Подключает **MCP-серверы** (function-calling, агентный цикл по нескольким серверам), голосовой ввод `/voice` (микрофон → текст через Yandex SpeechKit), ANSI-рендер markdown, подтверждения операций (MCP elicitation). При подключённом RAG (`rag-mcp`) — ответы с **обязательными источниками и дословными цитатами** + локальный гейт против галлюцинаций и режим «не знаю» при слабом контексте, опц. LLM-судья достоверности (День 24). Флаги — `--max-tokens`, `--stop`, `--json`, `--json-schema`, `--no-thinking`, `--watch` (фоновые уведомления планировщика). |
| [`contract-extractor/`](contract-extractor) | Извлечение реквизитов сторон из договоров аренды. Два CLI: `extract-json` (строгий JSON-массив) и `list-text` (человекочитаемый список). |
| [`contract-generator/`](contract-generator) | Генерация больших списков синтетических договоров аренды (для тестовых данных). Режим стоп-маркера и пакетный режим. |
| [`task-solver/`](task-solver) | Решение одной задачи четырьмя способами промптинга (простой, пошаговый, двухшаговый, панель экспертов) и оценка GLM, какое решение точнее. |
| [`hf-multi/`](hf-multi) | Параллельный вызов нескольких моделей HuggingFace на один промпт (по умолчанию — крупная, средняя и мелкая по числу параметров) и сравнение ответов. |
| [`mcp-probe/`](mcp-probe) | Пробный MCP-клиент для исследования любых MCP-серверов: подключается по stdio или Streamable HTTP (настройка через `.env`), печатает список инструментов или результат вызова инструмента. Песочница на официальном `@modelcontextprotocol/sdk` (осознанное исключение из правила «без сторонних SDK»). |
| [`mcp-client/`](mcp-client) | Адаптер MCP-серверов к абстракции `ToolSet` ядра: подключение к нескольким серверам (stdio/HTTP), агрегирование их инструментов (неймспейс `сервер__инструмент`), рантайм add/remove. `connectionFactory(onElicit)` переадресует серверные запросы подтверждения (elicitation) приложению. Даёт агентам `llm-cli` доступ к MCP-инструментам. На официальном `@modelcontextprotocol/sdk`. |
| [`yandex-ocr-mcp/`](yandex-ocr-mcp) | MCP-сервер распознавания текста на изображениях через Yandex Vision OCR: инструмент `recognize-text` (источник — файл/URL/base64), настройка через `.env`. Два транспорта: stdio и Streamable HTTP (деплой на VPS). На официальном `@modelcontextprotocol/sdk` (осознанное исключение). |
| [`scheduler-mcp/`](scheduler-mcp) | MCP-сервер планировщика отложенных/периодических задач с фоновым исполнением 24/7: `schedule_task`/`list_tasks`/`cancel_task`/`run_now`/`get_history`/… Исполнители `http_check`/`note`/`agent`(LLM)/`system_metrics`/`report`. Доставка результатов — pull (`poll_results` + `llm-cli --watch`) или Telegram. Два транспорта (stdio/HTTP), деплой на VPS. |
| [`filesystem-mcp/`](filesystem-mcp) | MCP-сервер файловой системы с «мягкой» песочницей (allow-list): `read_file`/`write_file`/`append_file`/`list_dir`/`delete_path`. Путь вне allow-list требует подтверждения пользователя (MCP elicitation); удаление — без рекурсии и без удаления корня. |
| [`places-mcp/`](places-mcp) | MCP-сервер поиска организаций рядом: инструмент `find_places` (координаты из `get_my_location`) — список с расстоянием/адресом/телефоном/часами. Переключаемые провайдеры `PLACES_PROVIDER=osm\|yandex` (по умолчанию OpenStreetMap Overpass — без ключей). |
| [`rag/`](rag) | RAG-индексатор: загрузка документов (локальная папка / GitHub / URL с обходом по ссылкам) → чанкинг (2 стратегии: `fixed` и `structural`) → эмбеддинги (`core.EmbeddingsClient`, по умолчанию локальный Ollama) → локальный JSON-индекс с метаданными. CLI `build`/`compare`/`query` (сравнение стратегий + косинус-ретрив). |
| [`rag-mcp/`](rag-mcp) | MCP-сервер RAG: инструмент `search_docs` ищет релевантные фрагменты в проиндексированных документах, **строя индекс на лету** (github/папка/URL) при отсутствии, с кэшем и несколькими индексами. Staged-конвейер с реранкингом/фильтрацией (День 23): префиксы задачи nomic, порог `minScore`, `rerank` (MMR/LLM), `rewrite` (expand/HyDE), трасса стадий «до/после» и переопределения аргументами. Отдаёт `chunk_id` и сигнал уверенности — основа для обязательных цитат/источников и режима «не знаю» на стороне `llm-cli` (анти-галлюцинации, День 24). Подключается к `llm-cli` через `/mcp add` — агент сам ищет по документам и отвечает по источникам. Импортирует `rag`+`core`. |

Связка «генератор → экстрактор» образует замкнутый цикл: `contract-generator`
пишет файл договоров, который напрямую читает `contract-extractor`.

## Требования

- Node.js **>= 22.6** (нативный запуск `.ts` и `fetch`; зависимостей в рантайме нет).
- Доступ к OpenAI-совместимому API: URL, имя модели, API-ключ (по умолчанию
  настроено на z.ai / GLM).

## Быстрый старт

Каждый прикладной пакет — самостоятельный: установка и `.env` свои.

```bash
cd llm-cli            # или contract-extractor / contract-generator
npm install
cp .env.example .env  # впишите LLM_API_KEY, LLM_BASE_URL, LLM_MODEL
npm start             # см. README конкретного пакета
```

`core/` отдельно ставить не нужно — он подключается из соседних пакетов по
относительному пути. Свои зависимости (`@types/node`, `typescript`, `prettier`)
есть у каждого пакета.

## Конфигурация (общие переменные окружения)

Читаются ядром из `.env` в каталоге запуска (переменные окружения имеют приоритет):

| Переменная | Назначение | По умолчанию |
| ---------- | ---------- | ------------ |
| `LLM_API_KEY` | Ключ доступа к API | — (обязательно) |
| `LLM_BASE_URL` | URL эндпоинта chat/completions | — (обязательно) |
| `LLM_MODEL` | Имя модели | — (обязательно) |
| `LLM_TEMPERATURE` | Температура | `0.7` |
| `LLM_SYSTEM_PROMPT` | Системный промпт | нейтральный |
| `LLM_REQUEST_TIMEOUT_MS` | Таймаут запроса, мс | `60000` |
| `LLM_MAX_RETRIES` | Число повторов при 429/5xx/сбоях | `3` |
| `LLM_RETRY_BASE_MS` | Базовая задержка бэкоффа, мс | `500` |
| `LLM_STAGE_MAX_TOKENS` | Потолок генерации агентов пайплайна задач, токенов | дефолт провайдера |
| `LLM_MAX_TOOL_ROUNDS` | Потолок раундов агентного цикла чата с MCP-инструментами (длинный кросс-серверный флоу) | `12` |
| `LLM_EMBEDDINGS_URL` | Эндпоинт `/embeddings` для RAG (`core.EmbeddingsClient`); напр. Ollama `http://localhost:11434/v1/embeddings` | — |
| `LLM_EMBEDDINGS_MODEL` | Модель эмбеддингов (напр. `nomic-embed-text`) | — |
| `LLM_EMBEDDINGS_API_KEY` | Ключ для эмбеддингов (для Ollama не нужен) | — |

## Разработка

В каждом пакете доступны одинаковые команды:

```bash
npm run typecheck   # проверка типов (tsc --noEmit)
npm test            # node:test, покрытие 100% (строки/ветки/функции)
npm run lint-prettier   # проверка форматирования
```

## Структура репозитория

```
ai-advent/
├── core/                  общее ядро (клиент, конфиг, типы, память, пайплайн)
├── llm-cli/               консольный чат с LLM (+ MCP, голос, задачи)
├── contract-extractor/    извлечение реквизитов из договоров
├── contract-generator/    генерация синтетических договоров
├── task-solver/           решение задачи разными стилями промптинга
├── hf-multi/              параллельный вызов моделей HuggingFace
├── mcp-probe/             песочница-исследование MCP-серверов
├── mcp-client/            адаптер MCP-серверов к ToolSet ядра
├── yandex-ocr-mcp/        MCP-сервер OCR (Yandex Vision)
├── scheduler-mcp/         MCP-сервер планировщика задач (24/7)
├── filesystem-mcp/        MCP-сервер файловой системы (песочница)
├── places-mcp/            MCP-сервер поиска организаций рядом
├── rag/                   RAG-индексатор документов (чанкинг + эмбеддинги + индекс)
└── rag-mcp/               MCP-сервер RAG (search_docs, индексация на лету)
```

Подробности по флагам и примерам — в README соответствующего пакета.
